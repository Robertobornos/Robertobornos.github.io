De Web App a
App Nativa
con Capacitor

Cómo convertí Mi Tienda —una app Ionic + Angular— en una aplicación nativa para Android integrando acceso a Cámara, GPS, Haptics y Share mediante los plugins de Capacitor.

👤 Roberto Pinto Gutiérrez 📚 Desarrollo de Aplicaciones Híbridas · 2DAW 📅 Junio 2026

Contexto técnico

¿Qué es Capacitor y por qué lo usamos?

Capacitor es el puente que permite que una aplicación web desarrollada con Ionic se ejecute como una aplicación nativa en Android e iOS. Actúa como una capa intermedia que envuelve el proyecto web en un contenedor nativo y expone las APIs del hardware del dispositivo —cámara, GPS, vibración— a través de plugins JavaScript.

A diferencia de soluciones antiguas como Cordova, Capacitor está diseñado para trabajar de forma nativa con los sistemas de build modernos y es el estándar recomendado por el equipo de Ionic Framework desde la versión 4.

🌐

Funciona en Web

Con PWA Elements, los plugins funcionan también en el navegador de escritorio para desarrollo.

📱

Nativo en Android

Genera un proyecto Android Studio completo con acceso real al hardware del dispositivo.

🔌

Plugins oficiales

@capacitor/camera, @capacitor/geolocation, @capacitor/haptics y @capacitor/share.

Plugins implementados

Los 4 plugins de hardware

Se han implementado dos plugins del reto básico y dos plugins adicionales para alcanzar la máxima calificación.

Básico

📷 @capacitor/camera

Acceso a la cámara del dispositivo. Devuelve la foto en formato base64 (DataUrl) para mostrarla directamente en un ion-img.

Básico

📍 @capacitor/geolocation

Obtiene las coordenadas GPS del dispositivo (latitud, longitud y precisión). Genera un enlace directo a Google Maps.

Extra

📳 @capacitor/haptics

Vibración táctil del dispositivo. Se usa como feedback al guardar nombre, tomar foto y añadir productos al catálogo.

Extra

📤 @capacitor/share

Abre el diálogo nativo de compartir del sistema operativo para compartir la foto tomada o la ubicación GPS obtenida.

Implementación paso a paso

Proceso de desarrollo

1

Instalar plugins y compilar

Se instalaron los 4 plugins de Capacitor y PWA Elements. Luego se compiló el proyecto con ionic build para generar la carpeta www/ necesaria para Capacitor.

2

Añadir plataforma Android

Con ionic cap add android se generó el proyecto nativo de Android Studio. Capacitor detectó automáticamente los 7 plugins disponibles.

3

Configurar AndroidManifest.xml

Se añadieron los permisos necesarios: CAMERA, READ/WRITE_EXTERNAL_STORAGE, ACCESS_FINE_LOCATION, ACCESS_COARSE_LOCATION e INTERNET. También se añadió usesCleartextTraffic="true" para el tráfico HTTP en desarrollo.

4

Crear servicios PhotoService y LocationService

Se encapsuló la lógica de cada plugin en servicios inyectables independientes, siguiendo el patrón de arquitectura establecido en la UT4.

5

Integrar en la página de Ajustes

Se añadieron las secciones Cámara, GPS y los botones de Share en la página de Ajustes. Los Haptics se integraron en guardar nombre, tomar foto y añadir productos.

6

Compilar y ejecutar en Android Studio

Con ionic build && npx cap sync && npx cap open android se abrió Android Studio, se sincronizó Gradle y se ejecutó la app en el emulador Pixel 7 API 36.

Permisos en AndroidManifest.xml

<!-- Permisos Cámara -->
<uses-permission android:name="android.permission.CAMERA" />
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"/>
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>

<!-- Permisos GPS -->
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />

<!-- Red + Cleartext HTTP -->
<uses-permission android:name="android.permission.INTERNET" />

Resolución de problemas

Errores encontrados y soluciones

Durante el desarrollo se encontraron varios errores técnicos que requirieron investigación y resolución.

ERROR: Could not find the web assets directory: ./www

Causa: Se ejecutó npx cap sync sin haber compilado el proyecto antes.
Solución: Ejecutar ionic build antes de npx cap sync. Capacitor necesita la carpeta www/ generada por el build de Angular.

ERROR: Cleartext HTTP traffic to localhost not permitted

Causa: Android bloquea por defecto el tráfico HTTP (no HTTPS) en apps nativas.
Solución: Añadir android:usesCleartextTraffic="true" en el tag <application> del AndroidManifest.xml.

ERROR: Camera plugin not working in browser (no UI)

Causa: Los PWA Elements no estaban inicializados antes de arrancar la aplicación.
Solución: Añadir defineCustomElements(window) en main.ts antes de bootstrapApplication().

npm error: could not determine executable to run (npx cap sync fuera de mi-tienda)

Causa: Los comandos npm/npx se ejecutaron en el directorio incorrecto.
Solución: Navegar siempre a la carpeta del proyecto con cd Desktop\DAH\mi-tienda antes de ejecutar cualquier comando.

Análisis de experiencia de usuario

Informe de usabilidad en el emulador

La app se probó en el emulador Medium Phone API 36 de Android Studio. A continuación se recoge el análisis de usabilidad comparando la experiencia web con la experiencia nativa.

✅ Aspectos positivos

  • La UI es idéntica en web y móvil gracias a los componentes Ionic adaptativos
  • El feedback táctil (Haptics) mejora notablemente la experiencia al confirmar acciones
  • La cámara se activa con permiso nativo y la foto se muestra en la app instantáneamente
  • El diálogo de Share nativo es familiar para el usuario y muy intuitivo
  • El ion-back-button se integra con la navegación nativa de Android
  • El dark mode se aplica automáticamente según el tema del sistema

⚠️ Aspectos a mejorar

  • En el emulador el GPS devuelve coordenadas simuladas, no reales
  • El catálogo aparece vacío porque el emulador no accede a localhost:3000
  • Los tiempos de carga del emulador son más lentos que en un dispositivo físico
  • La cámara del emulador muestra una escena virtual, no una cámara real
  • Algunos toasts quedan tapados por la barra de navegación del emulador

Comparativa Web vs Nativo

🌐

Versión Web

Cámara simulada con PWA Elements. GPS usando la API del navegador. Sin Haptics (no soportado en desktop). Share con Web Share API.

📱

Versión Nativa

Cámara real del dispositivo. GPS con alta precisión y chip dedicado. Haptics con motores de vibración físicos. Share con el diálogo nativo del SO.

Reflexión final

Conclusión

La conversión de Mi Tienda a aplicación nativa con Capacitor ha sido el paso más transformador del proyecto. Partir de una aplicación web funcional y, en pocos comandos, obtener una app Android con acceso real al hardware del dispositivo demuestra el poder del enfoque híbrido.

Lo más valioso ha sido comprobar cómo los patrones de arquitectura establecidos en unidades anteriores —servicios inyectables, componentes standalone, separación de responsabilidades— facilitan enormemente la integración de nuevas funcionalidades como los plugins nativos.

El proceso también ha reforzado la importancia de la resolución de problemas técnicos: cada error encontrado durante la configuración de Capacitor y Android Studio fue una oportunidad de entender más profundamente cómo funciona el puente entre el mundo web y el mundo nativo.
